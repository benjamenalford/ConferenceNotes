Classes

  • Syntactic sugar over prototype chain
  • It makes it feel like C# / Java
  • ... it still works the same

Classes

ES5: constructor function / prototype

function Person(name) {
    this.name = name;
}
Person.prototype.speak = function () {
    console.log('hello '+this.name;
};

ES6: class

class Person {
    constructor(name) {
        this.name = name;
    }
    speak() {
        console.log('hello '+name);
    }
}

Promise

Polyfill Asynchronous resolve

  • proxy for a later result
  • alternate style to callbacks
  • work is only done once for identical parameters
  • religious war: some find it more confusing

Promise

Polyfill Asynchronous resolve

ES5: callback

function doWork(cb) {
    cb(null, 'results');
}

doWork(function (err, results) {
    if (err) {
        // handle error
    }
    // handle success
});

ES6: promise

function doWork() {
    return new Promise(function (success, fail) {
        success('result');
    });
}

doWork().then(function (result) {
    // handle success
}).catch(err) {
    // handle error
});
